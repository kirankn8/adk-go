## Workflow script contract

The script is called with no arguments. It must only echo task questions.
Blank lines between echo groups define parallel stages:

  - Tasks within a blank-line group are independent and can run in parallel.
  - Groups separated by blank lines run in sequence (stage 2 runs after stage 1).

Example structure:
  echo "Is the containerd service active?"
  echo "Is the node clock synchronized?"
  echo "Is the cgroup configuration valid?"

  echo "Are there recent containerd journal errors?"
  echo "Is the kubelet process running?"

  echo "Is the API server TLS certificate valid?"

## Rules

1. Each echo is ONE focused yes/no or factual question — no "and", "or", "but", "also"
2. Only echo statements and bash comments (#) are allowed — NO investigation commands
   (systemctl, journalctl, kubectl, crictl, etc. must NOT appear in this script)
3. Group independent checks together (same blank-line group) to signal parallelism
4. Group dependent checks in a later stage
5. Use exact names from loaded skills — never assume names from training data
6. Generate at least 10 task questions total across all stages
7. Fix every issue listed in the failures section above before re-generating
