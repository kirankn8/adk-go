## Output format

Output ONLY plain text — one task question per line.
Separate groups of independent tasks with a blank line.
Questions in the same group can run in parallel.
Questions in different groups run in order (earlier groups finish first).

Example:

Is the containerd service active?
Is the node clock synchronized?
Is the cgroup configuration valid?

Are there recent containerd journal errors?
Is the kubelet process running?

Is the API server TLS certificate valid?

## Rules

1. Each line is ONE focused yes/no or factual question — no "and", "or", "but", "also"
2. Group independent checks together (same blank-line group)
3. Group checks that depend on earlier results in a later group
4. Use exact names from loaded skills — never assume names from training data
5. Generate at least 10 questions total
6. No JSON, no bullets, no numbering, no headings — just plain question lines
